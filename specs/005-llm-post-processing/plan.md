# Implementation Plan: LLM Post-Processing

**Branch**: `005-llm-post-processing` | **Date**: 2026-02-19 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/005-llm-post-processing/spec.md`

## Summary

Implement the LLM post-processing engine using llama.cpp (via `llama-cpp-2` 0.1 crate from utilityai). The engine takes raw ASR transcripts and produces polished text or structured voice commands using Qwen 2.5 3B Instruct (Q4_K_M quantization, ~1.6 GB on disk, ~2.2 GB VRAM). Token output is streamed via callback for low perceived latency. The model's built-in ChatML template handles prompt formatting, and `model.is_eog_token()` handles stop detection.

## Technical Context

**Language/Version**: Rust 2024 (1.85+)
**Primary Dependencies**: `llama-cpp-2` 0.1 (utilityai), `encoding_rs` 0.8 (new), `serde`/`serde_json` (workspace)
**Storage**: N/A (stateless inference engine)
**Testing**: `cargo test -p vox_core --features cuda` — unit tests (no model) + `#[ignore]` integration tests (require model)
**Target Platform**: Windows (CUDA), macOS (Metal)
**Project Type**: Existing three-crate Rust workspace (`vox`, `vox_core`, `vox_ui`)
**Performance Goals**: <100ms per 15-token output (RTX 4090), <275ms (M4 Pro), <2.2 GB VRAM
**Constraints**: Combined VRAM with Whisper ASR <6 GB, no network calls, local-only inference
**Scale/Scope**: Single-user dictation, one inference call at a time (sequential pipeline)
