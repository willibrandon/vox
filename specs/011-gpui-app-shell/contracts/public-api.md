## Module: `vox_ui::theme`

### `VoxTheme`

```rust
/// Shared visual theme accessible via `cx.global::<VoxTheme>()`.
///
/// Contains all color values for the Vox UI. Set as a GPUI Global during
/// app initialization. Components access colors via:
/// ```
/// let theme = cx.global::<VoxTheme>();
/// div().bg(theme.colors.surface)
/// ```
pub struct VoxTheme {
    pub colors: ThemeColors,
}

impl Global for VoxTheme {}

impl VoxTheme {
    /// Create the dark theme with all color values defined.
    pub fn dark() -> Self;
}
```

### `ThemeColors`

```rust
/// Complete color palette with 28 semantically-named colors.
///
/// All colors use GPUI's Hsla type. Organized by category:
/// backgrounds (4), text (3), borders (2), accent (2), status (6),
/// waveform (2), buttons (4), inputs (3).
#[derive(Clone)]
pub struct ThemeColors {
    // Backgrounds
    pub overlay_bg: Hsla,
    pub surface: Hsla,
    pub elevated_surface: Hsla,
    pub panel_bg: Hsla,

    // Text
    pub text: Hsla,
    pub text_muted: Hsla,
    pub text_accent: Hsla,

    // Borders
    pub border: Hsla,
    pub border_variant: Hsla,

    // Accent
    pub accent: Hsla,
    pub accent_hover: Hsla,

    // Status
    pub status_idle: Hsla,
    pub status_listening: Hsla,
    pub status_processing: Hsla,
    pub status_success: Hsla,
    pub status_error: Hsla,
    pub status_downloading: Hsla,

    // Waveform
    pub waveform_active: Hsla,
    pub waveform_inactive: Hsla,

    // Buttons
    pub button_primary_bg: Hsla,
    pub button_primary_text: Hsla,
    pub button_secondary_bg: Hsla,
    pub button_secondary_text: Hsla,

    // Input fields
    pub input_bg: Hsla,
    pub input_border: Hsla,
    pub input_focus_border: Hsla,
}
```

## Module: `vox_core::logging`

### Functions

```rust
/// Initialize structured logging with daily file rotation.
///
/// Creates the log directory if needed, sets up daily rotating file appender,
/// configures env-filter (VOX_LOG > RUST_LOG > default), and cleans up
/// logs older than 7 days. Returns a guard that must be held for the
/// application lifetime.
pub fn init_logging() -> LoggingGuard;

/// Platform-specific log directory path.
///
/// Windows: `%LOCALAPPDATA%/com.vox.app/logs/`
/// macOS: `~/Library/Logs/com.vox.app/`
pub fn log_dir() -> PathBuf;

/// Delete log files older than `retention_days` from the given directory.
///
/// Scans for files matching the `vox.YYYY-MM-DD.log` naming pattern and
/// removes those whose date stamp is older than `retention_days` from today.
/// Non-log files in the directory are left untouched.
pub fn cleanup_old_logs(dir: &Path, retention_days: u32);
```

### `LoggingGuard`

```rust
/// Guard that flushes pending log entries when dropped.
///
/// Must be held for the application lifetime. Store in a `let _guard`
/// binding in `main()`.
pub struct LoggingGuard {
    _guard: WorkerGuard,
}
```

## Module: `vox` (binary entry point)

### `main()`

```rust
/// Application entry point.
///
/// Execution sequence (FR-001 through FR-015):
/// 1. Initialize logging (creates log dir, sets up file rotation)
/// 2. Run GPUI Application
/// 3. Inside Application::run callback:
///    a. Create VoxState (settings, SQLite, dictionary) — set as Global
///    b. Create VoxTheme — set as Global
///    c. Register actions and key bindings
///    d. Open overlay HUD window (BEFORE model loading)
///    e. Set up system tray with context menu
///    f. Set up global hotkey for recording toggle
///    g. Spawn async pipeline initialization (model check/download/load)
///    h. Activate application
fn main();
```

### Internal Functions

```rust
/// Open the overlay HUD window with PopUp kind and transparent background.
fn open_overlay_window(cx: &mut App);

/// Set up the system tray icon and context menu.
fn setup_system_tray(cx: &mut App);

/// Register the global hotkey for recording toggle.
fn setup_global_hotkey(cx: &mut App);

/// Async pipeline initialization: check models, download if missing, load pipeline.
async fn initialize_pipeline(cx: AsyncApp);
```
